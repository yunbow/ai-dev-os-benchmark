# テスト 001: ガイドライン量の影響 — ガイドラインの量は AI コード品質に影響するか？

## 結果

**厳選10ファイル > ガイドラインなし > 全28ファイル。** ガイドラインは多いほど良いわけではない。

| 条件 | 平均スコア | ベースラインとの差 |
|------|:---:|:---:|
| A: ガイドラインなし | 84.1 | — |
| B: 全28ファイル（~75Kトークン） | 79.3 | **−4.8** |
| C: 厳選10ファイル（~24Kトークン） | 84.9 | **+0.8** |

**注意分散を確認:** 全ガイドラインをロードするとエラーハンドリングが 13.7 → 10.0 に低下。ガイドラインなしの方がうまくできることに AI が注意を払えなくなった。

これは AI Dev OS の [Two-Tier Context Strategy](https://github.com/yunbow/ai-dev-os#key-concepts) を実証 — 10-15の高インパクトルールを静的コンテキストにロードし、30以上は動的にオンデマンドチェック。

## セットアップ

- **アプリ:** タスク管理アプリ（5機能: 認証、CRUD、カテゴリ、チーム、API）
- **モデル:** Claude Opus 4.6
- **実行回数:** 条件ごとに3回
- **採点:** 9評価軸 × 66項目（100点満点）

## 検証された仮説

| # | 仮説 | 結果 |
|---|------|:---:|
| H1 | C > A（厳選 > ベースライン） | **確認**（+0.8、僅差） |
| H2 | C ≥ B（厳選 ≥ 全量） | **確認**（+5.6） |
| H3 | B < A（注意分散） | **確認**（−4.8、最も強い発見） |
