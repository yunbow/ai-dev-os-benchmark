# テスト 002: Before/After 例の効果 — コード例は AI の遵守率を改善するか？

## 結果

**改善なし。** security.md に Before/After コード例（+280行、+55%）を追加しても、ターゲット項目の遵守率は改善しなかった。

| タスク | D（例なし） | E（例あり） | 勝者 |
|--------|:---:|:---:|:---:|
| 1. プロフィール更新 | 全Pass | 全Pass (V2 Partial) | D |
| 2. 招待メール | 全Pass | 全Pass | 引き分け |
| 3. ログイン + レート制限 | 全Pass (+タイミング保護) | 全Pass | D |
| 4. タスク検索 API | 6/6 | 6/6 | 引き分け |
| 5. パスワードリセット | 5/5 (+トークンハッシュ) | 4/5 | **D** |

モデルは散文ガイドラインだけで十分にセキュアなコードを生成する。タスク5では条件Eが `escapeHtml()` を定義しながら呼び出さなかった — 原則の理解ではなく表面的なパターンコピー。

**判定:** ガイドラインに Before/After セクションを追加しない。インライン ❌/✅ 例は最小限（1-3行）に留める。
